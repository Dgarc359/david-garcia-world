Set up Gremlin Console​

Set up Empty Graph​

graph = TinkerGraph.open() g = traversal().withEmbedded(graph)

Load Sample Data into console​

Start gremlin console: bin/gremlin.sh

Loading up the "Modern Graph" -> Insert picture of graph <- graph = TinkerFactory.createModern()

Creating a TraversalSource g = traversal().withEmbedded(graph)

- Provides traversal strategies to appli and the traversal engine to use

Info on withEmbedded() https://tinkerpop.apache.org/docs/3.5.1/reference/#connecting-embedded

Traversing

g.V() //// (1)
g.V(1) //// (2)
g.V(1).values('name') //// (3)
g.V(1).outE('knows') //// (4)
g.V(1).outE('knows').inV().values('name') //// (5)
g.V(1).out('knows').values('name') //// (6)
g.V(1).out('knows').has('age', gt(30)).values('name')    // (7)
  1. Get all vertices in the Graph
  2. Get the vertex with unique id of '1'
  3. Get the value of the name property on the vertex with unique id '1'
  4. Get the edges with the label "knows" for the vertex with the unique id of "1"
  5. Get the names of the people whom the vertex with the unique id "1" "knows"
  6. Note: when using outE().inV() as shown in 5, this can be shortened to just out() similar to inE().outV() and in() for incoming edges
  7. Get the names of the people vertex '1' knows, IF they are over age 30

IMPORTANT A Traversal is an iterator, if you assign a variable to something like x = g.V() you will need to iterate over x yourself, the gremlin console automatically does this for you

Locating a specific vertex​

g.V().has('person', 'name', 'marko') Here, gremlin is iterating through all vertexes and locating a 'person' vertex that has the property 'name' with value 'marko'

IMPORTANT Locating a vertex in this fashion works for sample Databases, but more complex databases with millions or billions of vertices are going to be a bigger problem for this method. Gremlin recommends consulting documentation of the graph / graph db you have chose and utilize its native API to create indices which will then speed up these types of lookups.

Notes on valuemap / elementmap​

It may be more useful to use by(__.identity()) over by(elementMap())

Working with airport sample data

**Find all vertex labels:** `g.V().label().dedup()`
dedup makes sure we get a list of unique label names back

**Find all edge labels:** `g.E().label().dedup()`

**Find vertex properties:** `g.V().hasLabel('airport').limit(1).next().keys()`

---

**Airports with the most routes?**
`g.V().hasLabel('airport').where(out('route').count().is(gt(180))).group().by('code').by(out().count()).order(local).by(values,desc)`

**What is the distribution of runways in the graph?**
`g.V().hasLabel('airport').groupCount().by('runways')`

**Does any route exist between two airports?**
`g.V().has('code','AUS').repeat(out().dedup()).until(has('code','YPO')).path().by('code')`

**What airports in the US / Canada can I fly to from india?**
`g.V().has('country','code','IN').out().out().has('country',within('US','CA')).path().by('code')`

we can fold this and make it an array

**Find all the routes between cities x, y, and z**
`g.V().has('city', within('London','Munich','Paris')).aggregate('a').out().where(within('a')).path().by('code')`

`g.V().has('city', within('Miami','Atlanta','Paris')).aggregate('a').out().where(within('a')).path().by('code')`

---

`g.V().label().dedup()`

**Find weights and vertices associated with vertex**
`g.V().has('name','marko').bothE().otherV().path().by(__.valueMap(true))`

represent our data for a given time period =, all of the user data
