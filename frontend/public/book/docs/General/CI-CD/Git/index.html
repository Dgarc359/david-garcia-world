Standard Git Workflow​

  • git add .
  • git commit -m "your message"
  • [OPTIONAL - IF YOU MAKE A NEW GITHUB REPO] git remote add
  • git push

Clone a Git Repository​

Setting up Git with your local Computer​

Set up ssh key

 ssh-keygen -t ed25519 -C "<yourEmailHere>"

 eval "$(ssh-agent -s)"

 ssh-add ~/.ssh/id_ed25519
 
 ssh-add -l

Add ssh key to github

  1. Go to you github account settings
  2. Find 'ssh and GPG keys'
  3. Add new key
  4. run the following command cat ~/.ssh/id_ed25519.pub
  5. copy paste that into github

Any new repositories you make, you can now easily access without needing further authentication

Cheat Sheet​
  • Track remote branch and choose local name
    git branch --track local-branch-name remote-name/branch-name
  • Pull remote branch into local branch
    1. git remote add remote-name remote-url
    2. git fetch remote-name
    3. git checkout -b local-branch-name remote-name/remote-branch-name
  • Change default branch on git init to main
    git config --global init.defaultBranch main
  • Delete remote branch
    git push remote --delete branch
  • Change default git editor
    git config --global core.editor vim
  • Prettier git log
    git log --oneline --decorate

Git Squash / Fixup​

When to use Squash vs. Fixup Squash: you want your commit message to stay Fixup: you don't want your commit message to stay

The process:

$ git add .

$ git commit -m "message" --fixup :/commit-message 
    // or --squash, the :/ finds the latest commit containing your 'commit-message' and attaches the fixup or squashed commit to it
    
$ git rebase -i HEAD~2 --autosquash
    // change the integer after HEAD to include whatever commits you're squashing
    
$ git push --force
    // force push in order to rebase remote repo
